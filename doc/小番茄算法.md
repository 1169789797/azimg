# 小番茄算法规则说明

小番茄算法是一种**基于像素置换**的图片混淆方案，通过 Gilbert 空间填充曲线确定遍历顺序，再按固定偏移量重排像素位置。本项目的实现与 **xiaofanqie（小番茄）**、**PicEncrypt** 等官网工具保持一致。

---

## 一、算法概述

| 项目 | 说明 |
|------|------|
| 算法类型 | 可逆像素置换（非加密哈希，无信息丢失） |
| 遍历方式 | Gilbert 2D 空间填充曲线 |
| 密钥作用 | 控制像素沿曲线方向的循环偏移量 |
| 默认密钥 | `0`（等同黄金比例偏移） |
| 密钥范围建议 | `0 ~ 1.618` 之间的小数；`0` 或留空则使用黄金比例 |

混淆后的图片在视觉上呈现为「噪点/色块」效果；使用**相同密钥、相同宽高**执行反混淆即可还原。

---

## 二、核心规则

### 1. Gilbert 曲线遍历

算法首先根据图片宽高 `width × height` 生成 Gilbert 曲线，得到长度为 `pixelCount = width × height` 的坐标序列 `curve`：

- `curve[i]` 表示曲线上第 `i` 个像素点的 `(x, y)` 坐标
- 遍历顺序由 `gilbert2d(width, height)` 唯一确定
- 当 `width >= height` 与 `width < height` 时，曲线起始方向不同，但始终覆盖全部像素且顺序固定

**规则：** 同一尺寸的图片，曲线生成结果必须完全一致，否则无法与外部工具互通。

### 2. 偏移量（offset）计算

偏移量决定「原曲线位置 `i` 的像素」被移动到「曲线位置 `i + offset`」。

```
pixelCount = width × height
```

**有密钥时**（密钥字符串去首尾空格后非空）：

```
k = parseFloat(key)
若 k 为有效正数：
  offset = round(pixelCount × k / 1.618) % pixelCount
否则：
  回退到黄金比例（见下）
```

**无密钥时**（留空、仅空格、或无效密钥）：

```
offset = round(pixelCount × (√5 - 1) / 2)
```

其中 `(√5 - 1) / 2 ≈ 0.618`，即黄金比例的倒数 φ⁻¹。

**密钥与偏移对照示例**（以 4096 像素为例）：

| 密钥 | 偏移量计算 | offset |
|------|-----------|--------|
| 留空 | round(4096 × 0.618) | 2531 |
| `0` | 回退黄金比例（同留空） | 2531 |
| `1` | round(4096 × 1 / 1.618) % 4096 | 2531 |
| `0.618` | round(4096 × 0.618 / 1.618) % 4096 | 1564 |
| `0.5` | round(4096 × 0.5 / 1.618) % 4096 | 1266 |

### 3. 像素置换规则

对每个曲线序号 `i`（从 `0` 到 `pixelCount - 1`）：

```
oldPos = curve[i]                    // 原位置
newIdx = (i + offset) % pixelCount
newPos = curve[newIdx]               // 目标位置
```

**混淆（encrypt = true）：**

```
将 oldPos 处的 RGBA 像素 → 写入 newPos
```

**反混淆（encrypt = false）：**

```
将 newPos 处的 RGBA 像素 → 写回 oldPos
```

即混淆与反混淆互为逆操作，交换源索引与目标索引。

### 4. 像素数据格式

- 每个像素占 **4 字节**：R、G、B、A
- 索引计算：`index = 4 × (x + y × width)`
- **Alpha 通道一并参与置换**，不做单独处理

---

## 三、密钥规则

| 规则 | 详情 |
|------|------|
| 输入类型 | 字符串，内部用 `parseFloat` 解析 |
| 有效范围 | 大于 `0` 的数值；建议在 `0 ~ 1.618` 之间 |
| 留空行为 | 使用黄金比例偏移；密钥 `0` 与留空等效 |
| 无效密钥 | `NaN`、`≤ 0`、非数字 → 回退黄金比例 |
| 混淆与还原 | **必须使用完全相同的密钥**（包括是否留空） |
| 外部工具兼容 | PicEncrypt 等工具常用 `round(n × key / 1.618) % n` 公式 |

---

## 四、可逆性规则

1. **宽高不变**：混淆与反混淆必须在相同 `width`、`height` 下进行；改变尺寸后曲线与偏移均失效。
2. **密钥一致**：密钥必须完全相同，否则无法还原。
3. **单次可逆**：一次混淆 + 一次反混淆 = 原图（像素级完全一致）。
4. **多次操作**：连续多次混淆相当于用同一映射反复置换；偶数次且参数相同可还原。

## 六、算法流程图

```
输入 ImageData (width × height, RGBA)
        │
        ▼
生成 Gilbert 曲线 curve[0 .. pixelCount-1]
        │
        ▼
根据密钥计算 offset
        │
        ▼
对每个 i：
  oldPos ← curve[i]
  newPos ← curve[(i + offset) % pixelCount]
  按 encrypt 方向复制 RGBA 四通道
        │
        ▼
输出置换后的 ImageData
```

## 九、算法检测与噪点分析（实测）

以项目内 `原图.png`（1552×2232）与 `混淆后的图.jpg`（712×1024）为样本，通过加密对比、反混淆误差、管线模拟得出：

### 1. 检测到的算法变体

| 检测项 | 结论 |
|--------|------|
| 算法族 | **标准 Gilbert 曲线小番茄**（按曲线序号 `i` 遍历，非旧版网格查表） |
| 最匹配密钥 | **`1`** 或 **留空**（黄金比例偏移，两者 offset 仅差约 10 像素） |
| 排除项 | 密钥 `0.618` / `0.666` / `0.5`、旧版错误实现 — 匹配度显著更低 |
| LSB 隐写变体 | **未检测到**（若为 LSB 版，像素级误差模式会不同） |

### 2. 混淆图生成管线（推断）

对比多种「缩放 ↔ 加密 ↔ JPEG」顺序后，混淆图与以下管线最接近（MSE ≈ 78）：

```
原图 1552×2232 → 全尺寸小番茄混淆 → 缩放至 712×1024 → 保存 JPEG
```

而「先缩放到 712×1024 再混淆」与混淆图差异很大（MSE ≈ 236）。

**含义：** 你手里的 JPG 是在**较小尺寸**上的像素排列，但排列规则很可能来自**更大尺寸**（约 1552×2232）时的混淆结果再缩小，因此在 712×1024 上直接反混淆只能还原「大致轮廓」，无法与原图逐像素对齐。

### 3. 噪点 / 色差来源（算法本身无错）

| 原因 | 影响 | 说明 |
|------|------|------|
| **尺寸不一致** | 高 | 原图 1552×2232，混淆图 712×1024；反混淆必须在**混淆时的宽高**下进行 |
| **JPEG 有损压缩** | 中 | 混淆图存为 JPG，像素值被改写；模拟「加密→JPEG95→解密」仍有轻微误差，多次转发会更严重 |
| **缩放插值** | 中 | 全尺寸混淆后再缩小，缩放会破坏严格像素映射 |
| **算法变体错误** | 已排除 | 当前实现与 xiaofanqie / PicEncrypt 标准版一致 |

无损对照：同尺寸 PNG 加密 → 解密，**MSE = 0，100% 像素一致**。

### 4. 使用建议

1. 尽量使用**混淆工具直接导出的原文件**，不要经微信等平台二次压缩。
2. 反混淆时宽高必须与混淆时**完全一致**；若只有缩小版 JPG，可尝试先放大到疑似原始尺寸（如 1552×2232）再反混淆，但无法保证完美（JPEG + 插值已损伤）。
3. 密钥使用 **`0`** 或留空（黄金比例）。
4. 下载结果请选 **PNG**，避免再次 JPEG 压缩。
